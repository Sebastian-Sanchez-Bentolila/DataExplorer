Introducción a SQL

SQL (Structured Query Language) es un lenguaje estándar utilizado para gestionar y manipular bases de datos relacionales. Con SQL, puedes realizar diversas operaciones como consultas, inserciones, actualizaciones y eliminaciones de datos. A continuación, exploraremos los conceptos básicos de SQL, junto con ejemplos prácticos y salidas esperadas.

Historia y Evolución de SQL

SQL (Structured Query Language) es el lenguaje estándar para interactuar con bases de datos relacionales. Su desarrollo comenzó en la década de 1970 en IBM, donde se creó el sistema de base de datos relacional llamado System R. Este proyecto, basado en el modelo relacional propuesto por Edgar F. Codd en 1970, marcó el inicio de una nueva era en la gestión de datos.

SQL fue estandarizado por primera vez por ANSI en 1986 y por ISO en 1987. Desde entonces, ha evolucionado para incluir nuevas funcionalidades, como el manejo de datos jerárquicos, JSON, y soporte para análisis avanzado. Las versiones más recientes del estándar incluyen SQL:2016 y SQL:2019, que introducen capacidades modernas como análisis avanzado de datos y extensiones para big data.

Línea de Tiempo

  • 1970: Edgar F. Codd publica "A Relational Model of Data for Large Shared Data Banks".
  • 1974: IBM desarrolla el lenguaje SEQUEL, precursor de SQL.
  • 1986: ANSI establece el primer estándar de SQL.
  • 1990: Surgen sistemas comerciales como Oracle, MySQL y Microsoft SQL Server.
  • Presente: SQL es usado en casi todos los sistemas de bases de datos relacionales.

Conceptos Fundamentales de Bases de Datos Relacionales

Las bases de datos relacionales son sistemas diseñados para almacenar, gestionar y recuperar grandes cantidades de información estructurada. Este modelo se basa en el concepto de relaciones, representadas como tablas compuestas por filas y columnas.

Elementos Clave

  • Tablas: Representan entidades y están compuestas por filas (registros) y columnas (atributos).
  • Llaves Primarias: Identifican de manera única cada registro en una tabla.
  • Llaves Foráneas: Establecen relaciones entre tablas.
  • Consultas: Permiten recuperar datos específicos mediante el uso de SQL.

Creación de una Base de Datos y Tablas

Para comenzar a trabajar con SQL, primero necesitas crear una base de datos y luego crear tablas dentro de esa base de datos. Aquí tienes un ejemplo:


-- Crear una base de datos
CREATE DATABASE MiBaseDeDatos;
    
 -- Usar la base de datos creada
USE MiBaseDeDatos;
    
-- Crear una tabla llamada 'Estudiante'
CREATE TABLE Estudiante (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(50),
    Edad INT,
    Carrera VARCHAR(50)
);
    
-- Insertar datos en la tabla 'Estudiante'
INSERT INTO Estudiante (ID, Nombre, Edad, Carrera)
VALUES
(1, 'Ana', 20, 'Ingeniería'),
(2, 'Luis', 22, 'Matemáticas'),
(3, 'Carla', 21, 'Biología'),
(4, 'Jorge', 23, 'Física');

Salida en consola:


Query OK, 1 row affected (0.01 sec)
    
Query OK, 4 rows affected (0.00 sec)
Records: 4  Duplicates: 0  Warnings: 0

Consultas Básicas

Una de las operaciones más comunes en SQL es realizar consultas para recuperar datos de una tabla. Aquí tienes algunos ejemplos:

Seleccionar Todos los Datos


-- Seleccionar todos los datos de la tabla 'Estudiante'
SELECT * FROM Estudiante;

Salida en consola:


+----+--------+-----+------------+
| ID | Nombre | Edad | Carrera   |
+----+--------+-----+------------+
|  1 | Ana    |  20 | Ingeniería |
|  2 | Luis   |  22 | Matemáticas|
|  3 | Carla  |  21 | Biología   |
|  4 | Jorge  |  23 | Física     |
+----+--------+-----+------------+

Seleccionar Datos con Condición


-- Seleccionar estudiantes con edad mayor a 21
SELECT * FROM Estudiante WHERE Edad > 21;

Salida en consola:


+----+-------+-----+------------+
| ID | Nombre| Edad| Carrera    |
+----+-------+-----+------------+
|  2 | Luis  |  22 | Matemáticas|
|  4 | Jorge |  23 | Física     |
+----+-------+-----+------------+

Inserción, Actualización y Eliminación de Datos

Además de consultar datos, SQL te permite insertar nuevos datos, actualizar datos existentes y eliminar datos. Aquí tienes algunos ejemplos:

Inserción de Datos


-- Insertar un nuevo estudiante
INSERT INTO Estudiante (ID, Nombre, Edad, Carrera)
VALUES (5, 'María', 24, 'Química');

Salida en consola:


Query OK, 1 row affected (0.01 sec)

Actualización de Datos


-- Actualizar la edad de 'Ana'
UPDATE Estudiante SET Edad = 21 WHERE Nombre = 'Ana';

Salida en consola:


Query OK, 1 row affected (0.00 sec)
Rows matched: 1  Changed: 1  Warnings: 0

Eliminación de Datos


-- Eliminar el estudiante con ID 4
DELETE FROM Estudiante WHERE ID = 4;

Salida en consola:


Query OK, 1 row affected (0.00 sec)

Funciones Agregadas

SQL proporciona varias funciones agregadas que te permiten realizar cálculos en los datos. Aquí tienes algunos ejemplos:

Contar Filas


-- Contar el número de estudiantes
SELECT COUNT(*) AS TotalEstudiantes FROM Estudiante;

Salida en consola:


+----------------+
| TotalEstudiantes|
+----------------+
|               4 |
+----------------+

Calcular Promedio


-- Calcular la edad promedio de los estudiantes
SELECT AVG(Edad) AS EdadPromedio FROM Estudiante;

Salida en consola:


+-------------+
| EdadPromedio|
+-------------+
|          21 |
+-------------+

Uniones (JOINs)

Las uniones permiten combinar datos de dos o más tablas basadas en una condición relacionada. Aquí tienes un ejemplo:

INNER JOIN


-- Crear una nueva tabla 'Curso'
CREATE TABLE Curso (
    CursoID INT PRIMARY KEY,
    NombreCurso VARCHAR(50)
);
    
-- Insertar datos en la tabla 'Curso'
INSERT INTO Curso (CursoID, NombreCurso)
VALUES
(1, 'Matemáticas Avanzadas'),
(2, 'Física Cuántica'),
(3, 'Bioquímica');
    
-- Crear una tabla 'Inscripcion' que relaciona estudiantes con cursos
CREATE TABLE Inscripcion (
    EstudianteID INT,
    CursoID INT,
    FOREIGN KEY (EstudianteID) REFERENCES Estudiantes(ID),
    FOREIGN KEY (CursoID) REFERENCES Cursos(CursoID)
);
    
-- Insertar datos en la tabla 'Inscripcion'
INSERT INTO Inscripcion (EstudianteID, CursoID)
VALUES
(1, 1),
(2, 2),
(3, 3);
    
-- Realizar un INNER JOIN entre 'Estudiante' e 'Inscripcion' para obtener los cursos de los estudiantes
SELECT Estudiante.Nombre, Curso.NombreCurso
FROM Estudiante
INNER JOIN Inscripcion ON Estudiante.ID = Inscripcion.EstudianteID
INNER JOIN Curso ON Inscripcion.CursoID = Curso.CursoID;

Salida en consola:


+--------+---------------------+
| Nombre | NombreCurso         |
+--------+---------------------+
| Ana    | Matemáticas Avanzadas|
| Luis   | Física Cuántica     |
| Carla  | Bioquímica          |
+--------+---------------------+

Ejemplo de Estructura Relacional

Supongamos que tenemos dos tablas en una base de datos para una librería:

Tabla: Libros

LibroID Título Autor Precio
1 El Quijote Miguel de Cervantes 15.99
2 Cien Años de Soledad Gabriel García Márquez 12.99

Tabla: Ventas

VentaID LibroID Cantidad Fecha
101 1 2 2025-01-01
102 2 1 2025-01-02

Ejemplo de Consulta SQL


-- Consulta para obtener todos los libros vendidos
SELECT Libros.Título, Ventas.Cantidad, Ventas.Fecha
FROM Libros
JOIN Ventas ON Libros.LibroID = Ventas.LibroID;

Salida en consola:

Título Cantidad Fecha
El Quijote 2 2025-01-01
Cien Años de Soledad 1 2025-01-02
