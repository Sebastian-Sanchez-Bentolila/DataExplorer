Tipos de Datos en SQL

En SQL, los tipos de datos definen el tipo de valores que se pueden almacenar en una columna de una tabla. Los tipos de datos más comunes en SQL incluyen:

Tipo de Dato Descripción Ejemplo de Declaración
CHAR Cadenas de texto de longitud fija. Se rellenan con espacios si la longitud es menor a la especificada. 
columna CHAR(10)
VARCHAR Cadenas de texto de longitud variable. Solo almacena la cantidad real de caracteres. 
columna VARCHAR(50)
INT Números enteros. 
columna INT
FLOAT Números de punto flotante de precisión simple. 
columna FLOAT
DOUBLE Números de punto flotante de precisión doble. 
columna DOUBLE
DATE Fechas en el formato YYYY-MM-DD. 
columna DATE
TIME Horas en el formato HH:MM:SS. 
columna TIME
DATETIME Fecha y hora combinadas en el formato YYYY-MM-DD HH:MM:SS. 
columna DATETIME
TIMESTAMP Marca de tiempo que almacena fecha y hora en formato YYYY-MM-DD HH:MM:SS. Automáticamente actualiza la fecha y hora al insertar o actualizar registros. 
columna TIMESTAMP
BOOLEAN Valores lógicos. Puede almacenar TRUE o FALSE. 
columna BOOLEAN
TEXT Grandes cadenas de texto de longitud variable. 
columna TEXT
BLOB Grandes cantidades de datos binarios, como imágenes o archivos. 
columna BLOB

CHAR

El tipo CHAR (o CHARACTER) se utiliza para almacenar cadenas de texto de longitud fija. Se debe especificar la longitud máxima de la cadena al declarar una columna de tipo CHAR. Si la cadena almacenada es más corta que la longitud especificada, se rellenará con espacios.


CREATE TABLE ejemplo_char (
    columna1 CHAR(10)
);
Nota: CHAR(10) define una columna con una longitud fija de 10 caracteres.

VARCHAR

El tipo VARCHAR (o CHARACTER VARYING) se utiliza para almacenar cadenas de texto de longitud variable. A diferencia de CHAR, VARCHAR no rellena la cadena con espacios y solo almacena la cantidad de caracteres reales.


CREATE TABLE ejemplo_varchar (
    columna1 VARCHAR(50)
);
Nota: VARCHAR(50) define una columna que puede almacenar hasta 50 caracteres.

Salida en consola al insertar datos:


INSERT INTO ejemplo_varchar (columna1) VALUES ('Hola');
    
SELECT * FROM ejemplo_varchar;
/* Salida en consola:
columna1
-----------
    Hola        
*/

INT

El tipo INT (o INTEGER) se utiliza para almacenar números enteros. Es ideal para representar valores numéricos sin decimales.


CREATE TABLE ejemplo_int (
    columna1 INT
);
Nota: INT puede almacenar números enteros desde -2,147,483,648 hasta 2,147,483,647.

Salida en consola al insertar datos:


INSERT INTO ejemplo_int (columna1) VALUES (123);
    
SELECT * FROM ejemplo_int;
/* Salida en consola:
columna1
---------
    123
*/

FLOAT

El tipo FLOAT se utiliza para almacenar números de punto flotante de precisión simple. Es adecuado para almacenar valores numéricos con decimales y una amplia gama de valores.


CREATE TABLE ejemplo_float (
    columna1 FLOAT
);
Nota: FLOAT puede almacenar valores numéricos muy grandes y pequeños con decimales.

Salida en consola al insertar datos:


INSERT INTO ejemplo_float (columna1) VALUES (123.45);
    
SELECT * FROM ejemplo_float;
/* Salida en consola:
columna1
---------
    123.45
*/

DATE

El tipo DATE se utiliza para almacenar valores de fecha. El formato de la fecha es YYYY-MM-DD.


CREATE TABLE ejemplo_date (
    columna1 DATE
);
Nota: DATE almacena solo la parte de la fecha, sin información de la hora.

Salida en consola al insertar datos:


INSERT INTO ejemplo_date (columna1) VALUES ('2025-03-03');
    
SELECT * FROM ejemplo_date;
/* Salida en consola:
columna1
------------
    2025-03-03
*/

BOOLEAN

El tipo BOOLEAN se utiliza para almacenar valores lógicos. Puede tomar los valores TRUE o FALSE.


CREATE TABLE ejemplo_boolean (
    columna1 BOOLEAN
);
Nota: BOOLEAN almacena valores de verdad o falsedad.

Salida en consola al insertar datos:


INSERT INTO ejemplo_boolean (columna1) VALUES (TRUE);
    
SELECT * FROM ejemplo_boolean;
/* Salida en consola:
columna1
---------
    t
*/
