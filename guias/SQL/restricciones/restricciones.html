Restricciones en SQL

Las restricciones en SQL son reglas aplicadas a las columnas de una tabla para garantizar la integridad y validez de los datos almacenados. Estas restricciones evitan datos incorrectos o duplicados y aseguran relaciones consistentes entre las tablas.

Principales restricciones en SQL

NOT NULL

Impide que una columna tenga valores NULL. Se usa para asegurarse de que un campo siempre tenga un valor.


CREATE TABLE Clientes (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100) NOT NULL,
    Email VARCHAR(150) NOT NULL
);

En este caso, las columnas Nombre y Email no pueden tener valores nulos.

UNIQUE

Garantiza que los valores de una columna sean únicos en toda la tabla.


CREATE TABLE Productos (
    ID INT PRIMARY KEY,
    Codigo VARCHAR(50) UNIQUE,
    Nombre VARCHAR(100)
);

Aquí, el código de cada producto debe ser único.

PRIMARY KEY

Es una combinación de NOT NULL y UNIQUE. Se usa para identificar de manera única cada fila en una tabla.


CREATE TABLE Empleados (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Edad INT
);

La columna ID es clave primaria, lo que significa que su valor no puede repetirse ni ser NULL.

FOREIGN KEY

Define una relación entre dos tablas, asegurando que el valor en una columna coincida con un valor en la clave primaria de otra tabla.


CREATE TABLE Pedidos (
    ID INT PRIMARY KEY,
    ClienteID INT,
    FOREIGN KEY (ClienteID) REFERENCES Clientes(ID)
);

El ClienteID en la tabla Pedidos debe existir en la tabla Clientes.

CHECK

Restringe los valores permitidos en una columna.


CREATE TABLE Personas (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Edad INT CHECK (Edad >= 18)
);

Aquí, solo se permiten valores de edad mayores o iguales a 18.

DEFAULT

Asigna un valor por defecto a una columna si no se proporciona uno.


CREATE TABLE Usuarios (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Estado VARCHAR(20) DEFAULT 'Activo'
);

Si no se especifica un valor para Estado, se asignará 'Activo' por defecto.

INDEX

Mejora la velocidad de búsqueda en una tabla.

CREATE INDEX idx_nombre ON Clientes (Nombre);

Este índice acelera la búsqueda por nombre en la tabla Clientes.

AUTO_INCREMENT (MySQL) / SERIAL (PostgreSQL)

Permite generar valores automáticos en una columna, generalmente utilizada en claves primarias.


CREATE TABLE empleados (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100)
);

Restricciones en ALTER TABLE

También se pueden agregar o eliminar restricciones en tablas existentes con ALTER TABLE.

ALTER TABLE empleados ADD CONSTRAINT chk_edad CHECK (edad>= 18);
